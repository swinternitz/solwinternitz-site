Why I built it

I wanted a living-room friendly box that’s quiet, stable on the network, and doesn’t become a mess the moment you add a few hundred titles. The goal is “works every time” more than “tweak forever.”

Current status

  • Networking: wired Ethernet for bulk transfers; Wi‑Fi configured for stable remote access.
  • Fan control: MOSFET-driven GPIO fan with temperature thresholds (no constant fan chatter).
  • PS1: standardized to CHD; multi-disc handled via M3U with disc files hidden to keep the UI clean.

Networking & SSH (what was fixed)

Verified in the handoff notes (2025-12-27): wired Ethernet is preferred for bulk SCP transfers; Wi‑Fi IP stability is managed Pi-side. If Windows SSH warns that the host identification changed, clear the old known_hosts entry and re-accept.

Transfer workflow: use wired when moving lots of ROMs/CHDs; use Wi‑Fi for convenience once stable.

Fan control

The fan is controlled by GPIO through a MOSFET rather than directly off a header. The practical win is simple: the fan only ramps when it needs to, and it doesn’t oscillate on/off around a single threshold.

What I’m aiming for

  • Quiet at idle
  • No rapid on/off cycling
  • Predictable temperature bands

PS1 library hygiene: CHD + M3U

I’m standardizing PS1 to CHD. For multi-disc games, I use an M3U file as the visible entry, while disc CHDs are stored in a hidden folder so EmulationStation stays clean.

Folder pattern

psx/
  Game Title (Region).m3u
  .multidisc/
    Game Title (Disc 1).chd
    Game Title (Disc 2).chd
Why this matters: you get one visible game entry, correct disc switching, and no duplicate clutter.
